# Приёмник расшифровок для диктофона-часов

Записал на часах → остановил → аудио само ушло сюда → расшифровалось (ElevenLabs
Scribe) → текст (и само голосовое) прилетели тебе в личный **Telegram**.

Работает в облаке (Render, free), поэтому не зависит от того, включён ли Mac.

```
Часы (PWA, github.io/clock)  ──POST /upload──►  этот сервер (Render)
                                                   │
                                    ElevenLabs Scribe → текст
                                                   │
                                            Telegram (личный чат)
```

## Что нужно один раз настроить

### 1. Создать Telegram-бота
1. В Telegram напиши **@BotFather** → `/newbot` → задай имя. Получишь **токен**
   вида `1234567890:AAG...` → это `TELEGRAM_BOT_TOKEN`.
2. Открой своего нового бота и нажми **Start** (напиши ему любое сообщение) —
   иначе бот не сможет писать тебе первым.

### 2. Узнать свой chat_id
Подставь токен и открой в браузере:
```
https://api.telegram.org/bot<ТОКЕН>/getUpdates
```
В ответе найди `"chat":{"id": 123456789 ...}` — это число и есть
`TELEGRAM_CHAT_ID`.

### 3. Задеплоить сервер на Render
1. Залей папку `transcribe_server` в git-репозиторий (можно в тот же `clock`
   или отдельный).
2. Render → **New → Blueprint** → выбери репозиторий → он подхватит `render.yaml`.
3. Render спросит секреты — впиши:
   - `ELEVENLABS_API_KEY` — твой ключ ElevenLabs (тот же, что в speech-to-text).
   - `TELEGRAM_BOT_TOKEN` — из шага 1.
   - `TELEGRAM_CHAT_ID` — из шага 2.
   - `UPLOAD_SECRET` — придумай любую строку-пароль (напр. `myclock2026`).
4. Дождись деплоя. Адрес будет вида `https://clock-transcribe.onrender.com`.
   Проверь: открой этот адрес в браузере — должно ответить `ok`.

### 4. Включить выгрузку в самих часах
В `voice-recorder/index.html` вверху скрипта впиши:
```js
const UPLOAD_URL = 'https://clock-transcribe.onrender.com/upload';
const UPLOAD_KEY = 'myclock2026';   // тот же UPLOAD_SECRET
```
и передеплой часы: `./deploy_github.sh <github_token>`.

Готово. Дальше — только записывать.

## Поведение и нюансы
- **Когда отправляется:** в момент остановки записи, пока приложение открыто
  (iOS не даёт PWA работать в фоне). Если сети нет или сервер «спал» — запись
  остаётся в очереди и дошлётся при следующем открытии часов. В библиотеке:
  `✓` — отправлено, `↑` — ждёт отправки.
- **Холодный старт Render free:** первый запрос после простоя будит сервер ~30–50 сек.
  Часы это переживут (запрос просто подождёт), либо дошлют позже.
- **Локальная копия не удаляется** — всё аудио остаётся в самих часах как бэкап.
- **Переменные окружения** (Render → Environment):
  - `SEND_AUDIO=1` — слать в ТГ и само голосовое (0 — только текст).
  - `STT_LANGUAGE=rus` — язык расшифровки (пусто = автоопределение).
  - `UPLOAD_SECRET` — без него `/upload` открыт всем; задавай обязательно.

## Локальный запуск (для проверки на Mac)
```
cd voice-recorder/transcribe_server
ELEVENLABS_API_KEY=... TELEGRAM_BOT_TOKEN=... TELEGRAM_CHAT_ID=... \
UPLOAD_SECRET=test python3 server.py
```
Ключ ElevenLabs подхватится и из `../../speech-to-text/.env`, если не задан.
